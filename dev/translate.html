TRANSLATE

One limitation of css -transforms-
is that they are self-reversing.
This means that when you stop 
hovering (or clicking, etc) the 
effect reverses itself, like time
running backwards.  It would be 
nice if you could leave the items
where they were at end of the
transform.  Cannot though.

The 'translate' demo has a lot
of choreography in it. Every card
(and gold and text) must be 
positioned and its effect
timed to have it work out right.

The 'turned up' cards are in-place
all along - they are just 
-opacity: 0- until needed, then 
turned on with -opacity: 1.0-.
Since the face-up cards are loaded
after the blue-back cards, they
appear on top.  There is no need 
for z-index if you can load the
files in the proper order.

.cards {  /* common stuff for revealed cards */
  position: absolute;
  transform-origin: center center;
  transition: 1s ease-out .25s;
  opacity: 0;
}

   left: 135px;
   top: 375px;  }

article:hover #j-diamonds { /* reveal */
   opacity: 1.0;
   transition-delay: 6s;  }
